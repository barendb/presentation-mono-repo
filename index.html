lerna and yarn workspaces are tools to help you simplify these issues

   

BIG NOTE: they're not deployment tools

think very carefully how you will roll-out your code

  • what is the deployment cadence of each deployable package in your repo
  • what is your package versioning strategy (fixed or independent)
    • fixed all packages get the same version
    • independent package versions get incremented independently

whats our mono-repo deployment strategy?

  • platform code is in a mono-repo
  • production deploy => publish and fixed version increment
  • deploy entire stack every time
  • same version of code everywhere === less guess work
  • all automated - single button press
  • deploy very often, multiple times a day (when we get more devs ğŸ¤˜)
